# Module: Client (`client/`)

## Purpose
Primary user-facing desktop dashboard (Tauri v2 + Svelte 5). Users manage jobs, configure agents, view workers, manage projects. Connects to server via REST + WebSocket.

## Recent Updates (2026-03-24)

### Major Refactor: Client Performance Optimizations

**WS Log Batching (server-side)**: Server now batches WS log broadcasts at 200ms intervals, reducing message volume during high-throughput job streams.

**Job Store Performance**: `jobs.upsert()` now uses in-place `Object.assign` instead of spreading the entire array on every update. New `replaceAll()` method for bulk replacements. New `listStructureVersion` counter enables coarse-grained derivation — `workerOptions`, `bridgeOptions`, and `userOptions` derived stores only recompute when the job list structure actually changes (adds/removes), not on every property update.

**Autoscroll Performance**: Log autoscroll now uses `requestAnimationFrame` polling instead of `MutationObserver`, reducing layout thrashing during rapid log streaming.

## App Structure
- `App.svelte` - Root: TitleBar + (Legal Gate OR Boot Screen OR BootstrapWizard OR Setup OR Sidebar+Content+StatusBar) + Toast overlay. **First-run legal gate** blocks startup until user accepts pre-release disclaimer/no-warranty terms (persisted locally with versioned key). **Boot sequence**: checks saved `serverMode` - if `"local"`, auto-starts server and waits for "Server listening" in stdout before validating session; if `"remote"`, validates session immediately via `api.auth.me()`. ## Tauri Config (`src-tauri/`)
- `tauri.conf.json`: frameless window, custom titlebar, size 1200x800, bundle identifier `com.arkestrator.desktop`, and explicit updater plugin config (`plugins.updater.pubkey`, `plugins.updater.endpoints`) so updater plugin initialization is stable in dev/release config merges.
- `../scripts/ensure-sidecar.mjs`: validates expected sidecar binaries for the current host triple and triggers `pnpm --filter @arkestrator/server build:sidecar` if missing. It also ensures admin UI resources are bundled by copying `admin/dist` into `client/resources/admin-dist` before Tauri packaging/dev launch.
- `src/main.rs`: standard Tauri entry point
- `src/bridges.rs`: Bridge plugin management commands — `fetch_bridge_registry` fetches registry.json from GitHub Releases with 5-min TTL caching, `download_and_install_bridge` downloads and extracts zip to target path, `detect_program_paths` auto-detects DCC installations via glob hints, `get_installed_bridges`/`save_bridge_installation`/`uninstall_bridge` manage `~/.arkestrator/bridges.json` tracking, `check_path_exists` validates paths. Cross-platform path expansion for `~`, `%APPDATA%` etc. `detect_headless_programs` auto-detects installed DCC headless executables (Blender, Houdini hython, Godot) using `detect_executable_paths()` helper with cross-platform search patterns and `extract_version_from_path()` for version inference. Returns `DetectedHeadlessProgram` structs (program, path, version).
- `src/comfyui.rs`: ComfyUI process management commands — `detect_comfyui_paths` (cross-platform installation detection), `launch_comfyui` (launch as background process with managed PID), `stop_comfyui` (stop managed process), `is_comfyui_running` (check if managed process is alive), `check_comfyui_nodes` (check if arkestrator custom nodes are installed), `get_comfyui_autostart`/`set_comfyui_autostart` (read/write auto-start preference to config.json).
- `src/lib.rs`: custom Tauri commands - `write_shared_config` writes `~/.arkestrator/config.json` from the client's local machine, includes the canonical `workerName` plus persistent `machineId` for same-machine bridge reuse, auto-starts/restores a localhost bridge relay for remote servers so same-machine DCC bridges can target `127.0.0.1:<relay-port>` instead of the remote host directly, and now refuses malformed `apiKey` overwrites so a bad desktop-side value cannot clobber bridge auth during reconnect. The shared config preserves `remoteServerUrl` / `remoteWsUrl` for diagnostics and relay restore on next app launch. `get_machine_identity` prefers macOS `ComputerName`/`LocalHostName` via `scutil` before generic hostname fallback so worker registration uses the machine label users expect and generates a UUID-backed `machineId` persisted in shared config; `get_local_hardware_capability` returns native CPU/RAM/GPU hints for client-coordination capability probing; `list_local_ollama_models` probes desktop-local Ollama catalogs; `pull_local_ollama_model` performs desktop-local model download/update via Ollama CLI; `run_worker_headless` executes worker-local headless runs with timeout enforcement and script placeholder expansion so the desktop client can own Blender/Houdini/Godot CLI execution on behalf of the server. Filesystem commands: `fs_apply_file_changes` (batch apply file changes from delivery), `fs_create_directory`, `fs_write_file`, `fs_read_file_base64`, `fs_delete_path`, `fs_exists` — used by the `file_deliver` WS handler for cross-machine file delivery. HTTP transfer commands: `fs_download_file` (downloads a file from a server URL to a local path), `fs_upload_file` (uploads a local file to a server URL) — used by the `transfer_initiate` WS handler for HTTP-based file transfers. Depends on `base64 = "0.22"` crate. Registers plugins for shell, autostart, and updater; also sets up tray icon behavior (Show/Hide/Open Dashboard/Stop Server/Quit menu + close-to-tray window handling). "Open Dashboard" opens the admin page in the default browser (reads server URL from shared config, cross-platform via `open`/`cmd start`/`xdg-open`). "Stop Server" sends a POST to `/api/server/shutdown` on a background thread. ComfyUI integration: calls `comfyui::auto_start_comfyui_if_configured()` in `.setup()` hook and `comfyui::shutdown_comfyui()` on tray quit.
- `vite.config.ts`: injects `__CLIENT_VERSION__` plus `__CLIENT_BUILD__` (`<package version>+<git short sha>`) for visible in-app build display.

## Key UX Patterns
- Job list: compact rows with status dot, program icon (G/B/H), name or truncated prompt, worker name, short ID, relative time. Task-mode jobs show a `#T<N>` task badge in the row.
- Job detail panel: task-mode jobs display a progress bar (0-100%), task status text, and task ref (`#T<N>`) in the detail view
- Job tree: dependency-based nesting (children indented under parents)
- Resizable list panel (250px min, drag handle)
- Multi-select with checkboxes, bulk delete
- Status filter tabs (all/paused/queued/running/completed/failed/cancelled)
- "Start Queue" button resumes all paused jobs
